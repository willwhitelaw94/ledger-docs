# Context Documentation

This section documents TC Portal's product context, organized by:

- **[Customer Journey](/context/customer-journey)** — End-to-end experience from enquiry to ongoing care
- **[Domains](/context/domains)** — Core product areas you build (Budget, Care Plans, Claims, etc.)
- **[Concepts](/context/concepts)** — Business knowledge that informs the product (regulations, care pathways)
- **[Integrations](/context/integrations)** — External systems (Zoho, MYOB, Twilio, etc.)
- **[Personas](/context/personas)** — User personas and stakeholder profiles
- **[Industry](/context/industry)** — Aged care sector context, regulations, and reference material
- **[Government Resources](/context/government-resources)** — Converted PDFs and regulatory documents

## Domains vs Concepts

| | Domains | Concepts |
|---|---------|----------|
| **What** | Product features you build | Business knowledge that informs the product |
| **Examples** | Statements, Care Plan, Task Management | GST & Tax rules, Restorative Care Pathway |
| **Has code?** | Yes - UI, database, APIs | No - reference documentation |
| **Purpose** | Document how features work | Explain why features work that way |

## Creating Feature Documentation

When documenting a new domain or feature, use the template:

**[Domain Template →](/context/domains/_template)**

The template provides a consistent structure covering:

- Quick links, TL;DR, key concepts
- How it works (with mermaid diagrams)
- Business rules, feature flags
- Technical reference (models, actions, pages, API)
- Testing guidance and related domains
