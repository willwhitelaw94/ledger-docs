# Feature Specification: Fixed Asset Register & Depreciation

**Feature Branch**: `feature/033-FAR-fixed-asset-register`
**Created**: 2026-03-15
**Status**: Draft
**Input**: Idea Brief — toggleable asset register with automated depreciation/amortization via Financial Schedule Engine

## User Scenarios & Testing *(mandatory)*

### User Story 1 — Add an Asset (Priority: P1)

As a workspace user, I want to add a fixed or intangible asset to the register so I can track its value, depreciation, and lifecycle within my ledger.

When adding an asset, the user specifies: name, category (tangible/intangible), asset type (vehicle, equipment, property, furniture, plant, software, patent, goodwill, other), purchase date, purchase cost, residual/salvage value, useful life (years or months), depreciation method (straight-line, diminishing value, or units of production), and depreciation frequency (monthly, quarterly, annually). The system auto-creates two chart accounts: an asset account (e.g., "Vehicle — Toyota HiLux") and an accumulated depreciation contra account.

**Why this priority**: The asset record is the foundation — nothing else works without it.

**Independent Test**: Can be tested by adding a vehicle asset and verifying it appears in the register with correct details, linked chart accounts, and a depreciation schedule.

**Acceptance Scenarios**:

1. **Given** a workspace with the Assets module enabled, **When** a user creates a new tangible asset with type "Vehicle", cost $45,000, residual value $5,000, 5-year useful life, straight-line depreciation, monthly frequency, **Then** the asset appears in the register with status "In Service", two auto-created chart accounts (asset + accumulated depreciation), and a depreciation schedule showing 60 monthly entries of $666.67.
2. **Given** a user adding an intangible asset, **When** they select type "Software License" with a 3-year useful life, **Then** the asset is created with an amortization schedule using the same engine.
3. **Given** a user adding an asset, **When** they select "Units of Production" as the depreciation method, **Then** the system prompts for total estimated units and does not generate a time-based schedule — instead tracking depreciation per usage entry.

---

### User Story 2 — View Depreciation Schedule (Priority: P1)

As a user, I want to see the full depreciation schedule for an asset so I can understand how the value decreases over its useful life.

The asset detail page shows a schedule table with: period number, date, depreciation amount, accumulated depreciation, and net book value for each period. The schedule is generated by the Financial Schedule Engine from the asset's terms at creation.

**Why this priority**: The depreciation schedule is the core value — users need to see exactly how the asset's value will decline.

**Independent Test**: Can be tested by adding an asset with a 12-month useful life and verifying the schedule shows 12 rows with correct depreciation amounts and a final net book value equal to the residual value.

**Acceptance Scenarios**:

1. **Given** an asset with $12,000 cost, $0 residual, 12-month life, straight-line, **When** the user views the depreciation schedule, **Then** they see 12 rows of $1,000 each, with net book value decreasing from $12,000 to $0.
2. **Given** an asset with diminishing value method at 20% per year, **When** viewing the schedule, **Then** early periods show higher depreciation amounts that decrease over time.
3. **Given** a fully depreciated asset (net book value equals residual value), **When** viewing the schedule, **Then** all periods are marked as complete and no further depreciation is scheduled.

---

### User Story 3 — Automatic Depreciation Journal Entries (Priority: P1)

As a user, I want the system to automatically create depreciation journal entries each period so I don't have to manually post them.

The Financial Schedule Engine generates journal entries on each scheduled date: debit Depreciation Expense, credit Accumulated Depreciation. Journal entries are posted automatically when the scheduled period arrives. Users can review pending entries before they post, or configure auto-posting.

**Why this priority**: Automated JE generation is the key time-saver — the reason users adopt this module instead of doing it manually.

**Independent Test**: Can be tested by adding an asset with monthly depreciation and verifying that a journal entry is created for the current period with the correct accounts and amounts.

**Acceptance Scenarios**:

1. **Given** an asset with monthly depreciation of $500 due on the 1st of each month, **When** the scheduled date arrives, **Then** a journal entry is created: debit Depreciation Expense $500, credit Accumulated Depreciation $500, with the asset name in the description.
2. **Given** 5 assets all with monthly depreciation, **When** the scheduled date arrives, **Then** the system creates 5 separate journal entries (one per asset) for independent tracking.
3. **Given** a depreciation JE has been posted, **When** the user views the asset detail page, **Then** the corresponding schedule row is marked as "Posted" and the net book value reflects the depreciation.

---

### User Story 4 — Asset Register (Priority: P2)

As a user, I want to see all my assets in a single register so I can understand my total asset position at a glance.

The register shows: name, type, purchase date, original cost, accumulated depreciation, net book value, depreciation method, status, and remaining useful life. The register supports filtering by status (In Service, Disposed), asset category (Tangible, Intangible), and asset type. Summary row shows total cost, total accumulated depreciation, and total net book value.

**Why this priority**: The register is the primary navigation and overview point for assets.

**Independent Test**: Can be tested by adding 4 assets of different types and statuses, then verifying the register shows all with correct net book values and filters work.

**Acceptance Scenarios**:

1. **Given** a workspace with 6 assets (4 in service, 2 disposed), **When** the user opens the asset register, **Then** all 6 assets appear with current net book values, and the summary row shows totals.
2. **Given** the asset register, **When** the user filters by "Tangible" category, **Then** only tangible assets are shown with recalculated totals.
3. **Given** the asset register, **When** the user sorts by "Net Book Value" descending, **Then** the highest-value assets appear first.

---

### User Story 5 — Dispose an Asset (Priority: P2)

As a user, I want to dispose of an asset (sell, scrap, or write off) so the system calculates the gain or loss and creates the closing journal entries.

When disposing an asset, the user selects the disposal method (sold, scrapped, written off), enters the disposal date and proceeds amount (sale price, $0 for scrap/write-off). The system calculates: any remaining depreciation up to the disposal date, the gain or loss on disposal (proceeds minus net book value), and creates closing journal entries to remove the asset from the books.

**Why this priority**: Assets eventually leave the register — disposal is essential for accurate books.

**Independent Test**: Can be tested by disposing an asset with a known net book value and sale price, verifying the gain/loss calculation and closing JEs.

**Acceptance Scenarios**:

1. **Given** an asset with original cost $20,000, accumulated depreciation $12,000 (NBV $8,000), **When** the user sells it for $10,000, **Then** the system creates JEs: debit Bank $10,000, debit Accumulated Depreciation $12,000, credit Asset $20,000, credit Gain on Disposal $2,000. The asset status changes to "Disposed".
2. **Given** the same asset, **When** the user scraps it (proceeds $0), **Then** the system creates JEs recognising a $8,000 loss on disposal.
3. **Given** an asset disposed mid-period, **When** the disposal date is the 15th of the month, **Then** the system calculates pro-rata depreciation from the last posted period to the disposal date before calculating the gain/loss.

---

### User Story 6 — Asset Detail Page (Priority: P2)

As a user, I want to view an asset's full details so I can see its terms, depreciation schedule, posted journal entries, and current net book value in one place.

The detail page shows: asset summary (name, type, status, purchase date, cost, residual value, useful life, depreciation method), depreciation schedule tab (with posted/pending indicators), journal entry history tab (all depreciation JEs and any disposal/revaluation JEs), and a value summary (cost, accumulated depreciation, net book value, percentage depreciated).

**Why this priority**: Users need a comprehensive view to manage individual assets.

**Independent Test**: Can be tested by viewing an asset with 6 posted depreciation entries and verifying all sections show correct data.

**Acceptance Scenarios**:

1. **Given** an in-service asset with 12 posted depreciation JEs, **When** the user opens the detail page, **Then** they see the asset summary, 12 posted schedule entries, 12 linked journal entries, and correct net book value.
2. **Given** a disposed asset, **When** viewing its detail page, **Then** the disposal details (date, method, proceeds, gain/loss) are shown alongside the depreciation history.

---

### User Story 7 — Revalue an Asset (Priority: P3)

As a user, I want to revalue an asset up or down so I can reflect changes in market value or impairment.

When revaluing, the user enters the new carrying value and a reason. The system calculates the adjustment amount and creates a journal entry: for upward revaluation, credit Revaluation Reserve; for downward (impairment), debit Impairment Loss. The depreciation schedule recalculates from the new carrying value over the remaining useful life.

**Why this priority**: Revaluation is important for property and long-lived assets but not needed for basic depreciation tracking.

**Independent Test**: Can be tested by revaluing an asset and verifying the adjustment JE and recalculated depreciation schedule.

**Acceptance Scenarios**:

1. **Given** an asset with NBV $50,000 and 5 years remaining, **When** the user revalues it to $70,000, **Then** the system creates a JE (debit Asset $20,000, credit Revaluation Reserve $20,000) and recalculates future depreciation based on $70,000 over 5 years.
2. **Given** an asset with NBV $50,000, **When** the user impairs it to $30,000, **Then** the system creates a JE (debit Impairment Loss $20,000, credit Asset $20,000) and recalculates future depreciation.

---

### User Story 8 — Dashboard Asset Widget (Priority: P3)

As a user, I want to see an assets summary on my dashboard so I can quickly check total asset value and depreciation for the current period.

The widget shows: total original cost, total accumulated depreciation, total net book value, depreciation expense for the current period, and a count of assets by status.

**Why this priority**: Dashboard visibility adds polish but isn't essential for core tracking.

**Independent Test**: Can be tested by adding assets and verifying the widget shows correct totals.

**Acceptance Scenarios**:

1. **Given** a workspace with 8 in-service assets, **When** the user views the dashboard, **Then** the assets widget shows total cost, total NBV, and this month's total depreciation expense.
2. **Given** no assets, **When** the user views the dashboard, **Then** the widget shows an empty state with a prompt to add an asset.

---

### User Story 9 — Asset Reporting (Priority: P3)

As a user, I want to run an asset schedule report so I can see all assets with their depreciation status for financial reporting and tax preparation.

The report shows: each asset with original cost, accumulated depreciation, net book value, depreciation method, remaining useful life, and depreciation for the selected period. Totals by asset type/category. Exportable for accountant/auditor review.

**Why this priority**: Reporting supports year-end and tax preparation but isn't needed for daily use.

**Independent Test**: Can be tested by running the report and verifying totals match the register.

**Acceptance Scenarios**:

1. **Given** 10 assets across 3 types, **When** the user generates the asset schedule report for the current financial year, **Then** each asset shows year-to-date depreciation, and subtotals appear per asset type.
2. **Given** the report, **When** the user exports it, **Then** it downloads in a format suitable for external review.

### Edge Cases

- What happens when an asset reaches the end of its useful life but isn't disposed? Depreciation stops at the residual value; the asset remains in the register with status "Fully Depreciated" until manually disposed.
- What happens when the residual value is changed after depreciation has started? The remaining depreciation recalculates over the remaining useful life using the new residual value.
- What happens if a depreciation period is skipped (e.g., system was offline)? Catch-up entries are created for all missed periods when the system next runs.
- Can an asset be un-disposed? No — disposal is permanent. If disposed in error, create a new asset with the same details.
- What about assets with $0 cost (donated/gifted)? System allows $0 cost assets with optional fair value for depreciation purposes.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST allow users to create assets with: name, category (tangible/intangible), type, purchase date, cost, residual value, useful life, depreciation method, depreciation frequency, linked Contact (supplier/vendor), and linked purchase Bill or Journal Entry.
- **FR-002**: System MUST support asset types: Vehicle, Equipment, Property, Furniture, Plant, Software, Patent, Goodwill, Trademark, and Other (custom label).
- **FR-003**: System MUST support depreciation methods: Straight-line, Diminishing value, and Units of production.
- **FR-004**: System MUST auto-create two chart accounts per asset: an asset account and an accumulated depreciation contra account.
- **FR-005**: System MUST generate a depreciation schedule via the Financial Schedule Engine showing period, date, depreciation amount, accumulated depreciation, and net book value.
- **FR-006**: System MUST automatically create journal entries for each depreciation period (debit Depreciation Expense, credit Accumulated Depreciation) via the Financial Schedule Engine.
- **FR-007**: System MUST track asset lifecycle through statuses: In Service, Fully Depreciated, Disposed.
- **FR-008**: System MUST support asset disposal with methods: Sold, Scrapped, Written Off — calculating gain/loss and creating closing journal entries.
- **FR-009**: System MUST calculate pro-rata depreciation when an asset is disposed mid-period.
- **FR-010**: System MUST display an asset register showing all assets with current net book values, status, and depreciation progress.
- **FR-011**: System MUST display an asset detail page with summary, depreciation schedule, journal entry history, and value breakdown.
- **FR-012**: System MUST support asset revaluation (upward to Revaluation Reserve, downward to Impairment Loss) with schedule recalculation.
- **FR-013**: System MUST stop depreciation when net book value reaches the residual value.
- **FR-014**: System MUST prevent deletion of assets that have posted depreciation entries — only disposal is allowed.
- **FR-015**: System MUST be a toggleable module — only visible when enabled in workspace settings.
- **FR-016**: System MUST support intangible asset amortization using the same Financial Schedule Engine as depreciation.
- **FR-017**: System MUST handle $0 cost assets (donated/gifted) with optional fair value entry for depreciation.
- **FR-018**: System MUST create catch-up entries for any missed depreciation periods.
- **FR-019**: System MUST display a dashboard widget showing total cost, total net book value, and current period depreciation.
- **FR-020**: Financial Schedule Engine MUST run as a daily scheduled job, posting any depreciation/amortization JEs due on or before today, with automatic catch-up for missed periods.
- **FR-021**: System MUST link each asset to the Contact (supplier/vendor) it was purchased from.
- **FR-022**: System MUST link each asset to its purchase transaction (Bill or Journal Entry) for full acquisition audit trail.
- **FR-023**: System MUST follow the Invoicing permission pattern — owner/accountant full access, bookkeeper create/view, approver reviews disposals/revaluations, auditor/client read-only.
- **FR-024**: System MUST support bulk CSV import of existing assets with: name, type, purchase date, cost, residual value, useful life, depreciation method, and accumulated depreciation to date. The schedule engine calculates remaining depreciation from the import date.

### Key Entities

- **Asset**: The core entity representing a fixed or intangible asset. Key attributes: name, category (tangible/intangible), type, purchase date, cost (cents), residual value (cents), useful life (months), depreciation method, depreciation frequency, status, linked asset chart account, linked accumulated depreciation chart account, linked Contact (supplier), linked Bill or Journal Entry (purchase transaction).
- **Financial Schedule** *(shared with 032-LAT)*: A generic schedule of future journal entries. The asset's depreciation schedule is an instance of this entity, with schedulable type "asset" and the appropriate calculation method.
- **Asset Disposal**: Records the disposal event. Key attributes: disposal date, method (sold/scrapped/written off), proceeds (cents), net book value at disposal, gain or loss, linked closing journal entries.
- **Asset Revaluation**: Records a revaluation event. Key attributes: date, previous carrying value, new carrying value, adjustment amount, reason, linked journal entry.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Users can add a new asset and see its depreciation schedule in under 2 minutes.
- **SC-002**: Depreciation journal entries are auto-created each period with zero manual intervention after initial setup.
- **SC-003**: Asset register net book values match the corresponding chart account balances in the general ledger at all times.
- **SC-004**: Disposal gain/loss calculations are accurate to the cent, matching manual calculation.
- **SC-005**: 100% of depreciation calculations match standard accounting formulas for each method.
- **SC-006**: Dashboard widget displays total asset position within 1 second of page load.
- **SC-007**: Bulk import of 50 assets with accumulated depreciation completes in under 30 seconds with correct remaining schedules.
