================================================================================
INTERACTION FLOW SUMMARY
================================================================================

Variation A (Table + Side Peek):
1. Coordinator sees table of calls
2. Clicks row → Side panel opens with call details
3. Links package via inline search
4. Adds notes
5. Clicks "Complete Review"
6. Panel closes, row updates, click next row

Variation B (Kanban + Center Modal):
1. Coordinator sees board with status columns
2. Clicks card → Modal opens with full details
3. Reviews transcription, links package
4. Clicks "Complete Review"
5. Modal closes, card moves to Completed column
6. Clicks next card or drags to "In Review"

Variation C (List + Inline Expand):
1. Coordinator sees compact list of calls
2. Clicks row → Row expands with full details
3. Reviews, links, adds notes all inline
4. Clicks "Complete Review"
5. Row collapses and marks complete
6. Click next row to expand

LEGEND
================================================================================

Status Indicators:
○ Pending     - Not yet reviewed
● In Review   - Currently being reviewed
✓ Completed   - Review finished

Package Indicators:
→ HCP-2847    - Linked to package
⚠ Unlinked    - Needs package assignment

View Toggles:
⊞ Table       - Spreadsheet view
☰ List        - Compact stacked view
▦ Board       - Kanban columns

Actions:
[✓ Complete]  - Primary action button
[×]           - Close panel/modal
▶ / ▼         - Expand/collapse toggle
